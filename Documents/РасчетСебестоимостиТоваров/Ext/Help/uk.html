Розрахунок собівартості товарів

Документ призначений для розрахунку собівартості товарів за певний період по зазначеній організації (або декільком організаціям).

Особливості роботи з документом


Собівартість розраховується з початку місяця по дату документа включно.

При виробництві продукції даний документ розподіляє загальну суму витрат між декількома вихідними виробами і розподіляє витрати між продукцією і незавершеним виробництвом.

Документ рекомендується формувати в кінці кожного місяця, так як саме до цього моменту стають відомі всі суми витрат, з яких складається собівартість. Важливо, що за місяць може бути введений тільки один документ цього виду.

Для коректного розрахунку собівартості товарів документ формується в рамках робочого місця Закриття місяця в розділі Фінансовий результат і контролінг.

Розрахунок собівартості товарів проводиться по організації. Якщо розрахунок собівартості проводиться для групи організацій, то в цьому випадку, метод оцінки вартості повинен бути єдиним для всіх організацій.
Для організацій, які продають товари один одного (див. настройки передачі товарів між організаціям), розрахунок собівартості повинен проводитися спільно.

Варіанти розрахунку собівартості


У системі передбачено два варіанти розрахунку собівартості:

  • Попередній розрахунок собівартості- призначений для попередньої оцінки вартості товарів для оцінки даних про вартість товарів в оперативному режимі. При цьому розраховується середня вартість товарів. Це дозволяє отримати швидку оцінку даних в різних звітах.
  • Фактичний розрахунок собівартості- проводиться в кінці місяця. Фактична собівартість розраховується відповідно до того методу оцінки вартості товарів, який заданий для організації торгового підприємства (середня за місяць або ФІФО). При фактичному розрахунку собівартості автоматично розподіляються всі додаткові витрати і можна зробити оцінку фінансової діяльності всього підприємства.


Для отримання попередньої оцінки собівартості протягом місяця використовується регламентне завдання, що запускається за налаштованим розкладом. Для цих цілей використовується настройка Розраховувати попередню собівартість регламентним завданням в розділі НДІ і адміністрування- Фінансовий результат і контролінг- Облік товарів.
При цьому попередня собівартість буде розраховуватися як середня за місяць, незалежно від того, який варіант розрахунку собівартості встановлений для конкретної організації.

Дані за попередньою собівартості можна отримати у звітах, призначених для аналізу фактичної собівартості (наприклад, Аналіз собівартості товарів). Також при попередньому розрахунку не проводиться запис даних, необхідних для коректного розрахунку собівартості в наступних періодах. Проте, попередній варіант розрахунку виконується швидше, особливо на великих обсягах даних.
