Настройка підтримання запасів

Робоче місце призначене для встановлення методів забезпечення номенклатури. Можливість використання робочого місця визначається функціональною опцією Забезпечення потреб-Розширене в розділі НДІ і адміністрування–Склад і доставка-Забезпечення потреб.

Порядок роботи по установці параметрів забезпечення:

  • Встановлення відбору за складом, на якому настроїти підтримання запасів.
  • Додавання товарів, для яких необхідно підтримувати запас.
  • Визначення параметрів підтримання для рядків таблиці (можливе встановлення відразу для декількох рядків). Також можна задати однакову кількість запасу для різних рядків - мінімальне, максимальне, страхове; можливо також задати норму споживання для рядків з відповідним методом забезпечення.
  • Для встановлення однакових параметрів забезпечення для декількох позицій передбачено довільна групування позицій в списку.

Діючі параметри. В даному режимі роботи відображається список підтримуваних запасів, а також характеристики, для яких запас не підтримується, але підтримується запас для номенклатури в цілому.

Рекомендації зі зміни. В даному режимі виводиться список товарів, для яких система рекомендує змінити підтримання запасу. Програма рекомендує підтримувати запас, якщо статистика витрачання товару відома і відхилення від розрахованого споживання не більше 5%, і навпаки не підтримувати, якщо статистика витрачання невідома або відхилення більше 5%. Середньоденне споживання і відсоток відхилення можна побачити в крайніх правих колонках таблиці.

Параметри розрахунку статистики споживання задаються за допомогою команди Настройка розрахунку статистики споживання. Розрахунок статистики відбувається або регламентним завданням, розклад запуску якого задається адміністратором системи, або, якщо регламентне завдання відключено, фоновим процесом при завданні відбору за складом.

Зміна підтримуваних запасів


Для додавання товарів до списку підтримуваних запасів доступні наступні команди:

  • Підбір товарів Додавання відбувається пономенклатурно зі списку товарів. Для номенклатури, у якій включений облік по характеристиках, додадуться загальні параметри підтримання запасу для номенклатури в цілому, що діють для всіх характеристик без індивідуальних параметрів.
  • За відбором. Додавання відбувається за обраними критеріями. Для номенклатури, у якій включений облік по характеристиках, додадуться індивідуальні параметри підтримання запасу без загальних параметрів для номенклатури в цілому.
  • Завантаження з зовнішнього файлу. Додавання відбувається за зіставленням даних з файлом. При завантаженні можна відразу задати параметри мінімального і максимального запасу.

Для обраної номенклатури в списку, у якій включений облік по характеристиках, можлива індивідуальна настройка (скасування настройки) параметрів підтримання запасу для використовуваних характеристик за допомогою відповідних команд в меню Групування.

Для вилучення товарів зі списку підтримуваних запасів призначена команда Виключити з підтримуваних запасів. При вилученні товару, для якого також налаштовані загальні параметри підтримання запасу для номенклатури в цілому, товар не зникне зі списку, а стане неактивним.

Параметри підтримання запасу


Методи підтримання запасу визначають, в який момент часу програма буде пропонувати сформувати замовлення номенклатури, і яку кількість буде запропоновано до замовлення:

  • Замовлення під замовлення - при даному методі забезпечення до замовлення буде пропонуватися тільки кількість, необхідна для відвантаження за замовленням.
  • Min - max - при даному методі необхідно вручну задати мінімальний і максимальний запаси товару. При зниженні залишку товарів до мінімального запасу програма буде пропонувати до замовлення кількість, що дорівнює максимальному запасу. Статистика продажів при цьому методі не враховується.
  • Розрахунок за статистикою - при даному методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше ніж на строк виконання замовлення за способом забезпечення. При цьому денне споживання буде розраховане за статистикою минулих періодів. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.
  • Розрахунок за нормою - при даному методі буде запропоновано сформувати замовлення, коли поточного залишку товарів (за вирахуванням страхового) буде вистачати не більше ніж на строк виконання замовлення за способом забезпечення, заданим для даного товару. Денна норма споживання при цьому методі задається вручну. Кількість до замовлення за цим методом розраховується таким чином, щоб товару вистачило на заданий строк забезпечення або до зазначеної дати.

При методах Підтримання запасу (..), якщо кількість, необхідна для відвантаження за замовленнями, буде перевищувати кількість, необхідну для підтримки складського запасу, автоматично буде запропоновано замовити кількість, необхідну для відвантаження за замовленнями.

Страховий запас задається тільки при методах забезпечення Розрахунок за нормою і Розрахунок за статистикою. Страховий запас дозволяє покрити можливі сплески попиту, в порівнянні з розрахованою статистикою, або забезпечити запас товарів при зриві строку поповнення запасу.

При підтримці запасу враховуються також потреби по замовленнях до відвантаження. Такі потреби фіксуються в замовленнях клієнтів і внутрішніх замовленнях за рахунок установки для замовлених позицій дії До забезпечення або Забезпечувати відокремлено. Настройка Потреби замовлень до відвантаження визначає яку кількість буде запропоновано до замовлення в залежності від сумарних потреб за замовленнями до відвантаження.

  • Забезпечувати за рахунок запасів-якщо кількість товарів у замовленнях з дією До забезпечення перевищує величину, що замовляється для підтримання запасу, то замовляється максимальна з двох величин.
  • Забезпечувати незалежно від підтримання запасів - кількість до замовлення складається з кількості для підтримання запасу і кількості в замовленнях з дією До забезпечення.

Відокремлені замовлення завжди забезпечуються незалежно від підтримання запасів.
