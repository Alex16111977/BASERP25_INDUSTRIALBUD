Настройка правил отримання фактичних даних бюджетування

Робоче місце призначене для настройки правил отримання фактичних даних бюджетування.

Правила використовуються для:

  • формування фактичних оборотів за Статтями бюджетів та залишків за Показниками бюджетів.
  • отримання фактичних даних при настроєному заповненні планових даних Примірника бюджету.


Фактичні дані не зберігаються і не вводяться в підсистемі бюджетування, а обчислюються по настроєним правилам у звітах і при автозаповненні планових статей. Для використання фактичних даних в настройці видів бюджетів існує напередвизначений сценарій Фактичні дані. Відповідність аналітик статтям і показниками бюджетування визначається автоматично за типами даних в джерелі фактичних даних.

Джерелами фактичних даних є:

  • Господарські операції та Статті активів/пасивів;
  • Залишки та обороти по рахунках регламентованого та міжнародного обліків;
  • Довільні дані (наприклад, регістри відомостей і довідники), отримання яких настроюється у вигляді схеми компонування даних.


Накстройка правил може бути здійснена в двох режимах: За статтями і показниками і За джерелами отримання даних.
У першому випадку, з яких джерел будуть сформовані фактичні дані за статтями і показниками. У другому випадку, за якими статтями і показниками джерела будуть сформовані фактичні дані.

Правила отримання фактичних даних бюджетування налаштовуються для бюджетних статей і показників. Для показників бюджету ці правила є джерелом отримання початкового сальдо, для статей бюджету - оборотів. Для статей бюджетів правила можуть бути налаштовані як для власне отримання фактичних оборотів за статтею, в цьому випадку вказується напередвизначений сценарій Фактичні дані, так і для зазначення правил для контролю лімітів витрачання ГК, для цього застосовується сценарій Виконання бюджету.

Перевірка результату роботи правил виконується за допомогою звіту Результат роботи правил. Звіт показує фактичні дані, які будуть отримані згідно з установленими правилами.

Якщо є правила, у яких встановлено прапорець Зберігати результат виконання правила - то за такими правилами фактичні дані зберігаються в регістрі накопичення Фактичні дані бюджетування. Для відображення документів в регістрі можна настроїти розклад за допомогою команди Настроїти розклад. Для примусового відображення документів слід використовувати команду Відобразити документи.

Особливості при настройці довільного доступу до даних ІБ

Для настройки правила отримання фактичних даних бюджетування за довільними даними інформаційної бази необхідно настроїти схему компоновки джерела даних.

Схема компонування даних повинна містити наступні поля:

  • Період даних. Поля періоду повинні називатися: Период, ПериодДень, ПериодНеделя…
  • Сума фактичного обороту (залишку). Сума може вибиратися у вихідній валюті, валюті управлінського обліку або у валюті регламентованого обліку. Поля повинні відповідно називатися: СуммаВВалюте, СуммаУпр, Сумарегл. Якщо вибирається сума у вихідній валюті, то в схемі має бути присутнім поле Валюта.


Схема може містити поля:

  • Аналитики (Контрагент, Номенклатура, Каса…). Для даних полів повинен бути вказаний Тип значення


Для обмеження періоду обираних даних використовувати параметри НачалоПериода і КонецПериода.
