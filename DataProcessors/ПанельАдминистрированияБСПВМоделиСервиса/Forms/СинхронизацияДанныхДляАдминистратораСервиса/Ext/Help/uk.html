У даній панелі здійснюється настройка синхронізації даних для інформаційної бази.

  • Каталог тимчасових файлів Windows
    		•  Шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Windows.
  • Каталог тимчасових файлів Linux
    		•  Шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Linux.

    Каталоги тимчасових файлів можна не заповнювати, коли в кластері BAF працює тільки один сервер або програма працює у файловому режимі.
    Якщо всі сервери кластера працюють під управлінням операційних систем одного сімейства (Windows/Linux), то друге поле заповнювати не обов'язково.

    Каталог повинен бути доступний на читання і запис для всіх серверів кластера. В тому числі - для всіх користувачів операційних систем, від імені яких запускаються сервери платформи BAF. Так само UNC шлях до нього повинен бути однаковим для всіх серверів одного сімейства - для цього каталог необхідно підключати до серверів під одним ім'ям.

    У полі Каталог тимчасових файлів Linux можна зазначити лише локальний шлях, наприклад /home/path.

    Щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора (root) зробіть наступне:

    mkdir folder
    --- замість folder напишіть ім'я каталогу. наприклад, в каталозі /home ми робимо mkdir shared - створить каталог shared з повним шляхом /home/shared.

    mount -t cifs //server/share folder -o //domain/user, password=pwd
    - де //server/share - UNC шлях до іншого комп'ютера, folder - шлях до каталогу на Linux комп'ютері, наприклад \home\shared , //domain/user - ім'я домену/користувача, при необхідності, pwd - пароль користувача (при необхідності).

    Якщо ім'я та пароль не потрібні, команда виглядає так:
    mount -t cifs //server/share folder

    Після цього у шляху до тому вказуємо folder, наприклад, /home/shared
