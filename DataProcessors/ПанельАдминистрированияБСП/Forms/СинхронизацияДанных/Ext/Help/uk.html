У цій панелі здійснюється організація спільної роботи у розподіленій інформаційної базі.

  • Синхронізація даних - увімкніть прапорець, щоб стати доступними розділ програмиСинхронізація даних та команди:
    • Настройка синхронізації даних– перехід до організації спільної роботи у розподіленій інформаційній базі.
    • Попередження при синхронізації даних- перехід до перегляду попереджень, що виникли при синхронізації даних (непроведених документів, незаповнених реквізитів, конфліктів та неприйнятих за датою заборони даних).
    • Префікс цієї інформаційної бази - використовується для визначення програми, у якій було введено елементи списків і документи. До кодів елементів списків та номерів документів автоматично додається префікс інформаційної бази.

Каталог тимчасових файлів кластера серверів BAF

  • Шлях до єдиного мережного каталогу для загальних тимчасових файлів серверів BAF, що працюють в одному кластері:
    • Під керуванням ОС Windows - вкажіть шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, які працюють під керуванням ОС Windows.
    • Під керуванням ОС Linux - вкажіть шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під керуванням ОС Linux.
  • Каталоги тимчасових файлів можна не заповнювати, коли в кластері BAF працює лише один сервер або програма працює у файловому режимі.
  • Якщо всі сервери кластера працюють під керуванням операційних систем одного сімейства (Windows/Linux), друге поле заповнювати не обов'язково.
  • Каталог повинен бути доступний на читання і запис для всіх серверів кластера. У тому числі для всіх користувачів операційних систем, від імені яких запускаються сервери платформи BAF.
  • UNC шлях до каталогу має бути однаковим всім серверів одного сімейства - для цього каталог необхідно підключати до серверів під одним ім'ям.
  • У полі Каталог тимчасових файлів Linux можна вказати тільки локальний шлях, наприклад, /home/path.
  • Для того щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора (root) зробіть наступне:
    • mkdir folder
      --- замість folder напишіть ім'я каталогу. наприклад в каталозі /home робимо mkdir shared - ця команда створить каталог shared з повним шляхом /home/shared.
    • mount -t cifs //server/share folder -o //domain/user, password=pwd
      - де //server/share - UNC шлях до іншого комп'ютера , folder - шлях каталогу на Linux комп'ютері, наприклад \home\shared, //domain/user - ім'я домену/користувача, якщо потрібно, pwd - пароль користувача (якщо потрібно).
    • Якщо ім'я та пароль не потрібні, команда виглядає так:
      mount -t cifs //server/share folder
    • Після цього на шляху до каталогу вказуємо folder, наприклад /home/shared.
