Схема забезпечення

Схема забезпечення призначена для встановлення основного способу забезпечення потреб на одному чи декількох складах, для цілей планування забезпечення. Настроєна схема забезпечення може бути призначена номенклатурній позиції або виду номенклатури.

Можливість використання схем забезпечення з'являється при включеній функціональної опції Розширене в розділі НДІ і адміністрування – Склад і доставка - Забезпечення потреб.

При включеній функціональній опції Управлінням виробництвом (версія 2.5) в розділі НДІ і адміністрування – Виробництво схеми забезпечення використовуються також для зазначення складів, з яких виробничі підрозділи повинні отримувати матеріали і на які передаються отримані вихідні вироби.

Якщо процес забезпечення потреб включає декілька послідовних етапів, коли номенклатурні позиції надходять на один склад, потім переміщуються на інші склади і т.д., доречно настроїти схему забезпечення. Така схема забезпечення буде включати в себе інформацію про способи забезпечення потреб кожного етапу (кожного задіяного складу).

При включеній функціональної опції Формати магазинів у розділі НДІ і адміністрування – CRM і маркетинг - Маркетинг є можливість встановити основний спосіб забезпечення потреб на всі магазини (склади) одного формату.

Якщо спосіб забезпечення потреб в окремій номенклатурній позиції або у її характеристиці на складі відрізняється від способу, встановленого за схемою забезпечення, то такий спосіб можна встановити для позиції безпосередньо на робочому місці Настройка способів забезпечення потреб,

Для встановлення способів забезпечення потреб передбачено два режими відображення складів: за форматами та за ієрархією. Список За форматами необхідно використовувати для установки способу забезпечення для всіх магазинів (складів), які належать одному формату.

Передбачено індивідуальне зазначення способу забезпечення для вибраного рядка списку, а також групове заповнення способу забезпечення по команді Заповнити спосіб забезпечення.

Якщо склад не задіяний в схемі забезпечення, виключити його зі схеми забезпечення дозволяє команда Очистити спосіб забезпечення.

Якщо включено управління виробництвом версії 2.5, на формі з'являється закладка Виробничі підрозділи. На цій закладці представлений список виробничих підрозділів, для яких в окремій колонці можна вказати склад.

Передбачено як індивідуальне зазначення складу для вибраного рядка списку, так і групове заповнення за командою Заповнити склад.

Обраний для підрозділу склад буде використовуватися при формуванні етапу виробництва. Для матеріалів заповнюється склад, з якого матеріал передається у виробництво. Для вихідних виробів заповнюється склад-одержувач.
