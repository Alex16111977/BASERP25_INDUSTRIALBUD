Настройки торгового представника

Визначає настройки користувачів мобільної прикладної програми (торгівельних представників): відбір даних, що беруть участь в обміні, а також деякі параметри управління обміном.

Як вказати торгового представника?

Інформація про торгових представників, які використовують при роботі мобільних пристроїв, вибирається з довідника "Користувачі". Тобто торговий представник, використовує у своїй роботі мобільні пристрої обов'язково повинен бути зареєстрований в довіднику "Користувачі".

Як швидко ввести дані про мобільний пристрій, для якого налаштовується обмін?

Для швидкого введення інформації про мобільний пристрій можна вказати його серійний номер. Передбачено також вибір мобільного пристрою зі списку раніше зареєстрованих мобільних пристроїв.

Як вказати версію мобільної прикладної програми, яка використовується при обміні?

Актуальна версія мобільної прикладної програми вибирається з довідника "Мобільні прикладні програми та їх версії".

Чи можна при обміні даними з мобільними комп'ютерами обмежити список переданої інформації?

При настройці правил обміну для кожної мобільної прикладної програми можна задати конкретні параметри відбору даних: за організацією, за номенклатурними позиціями, за клієнтами (партнерами) торгового підприємства, а також за складами, видами цін і причинами скасування замовлень. Встановлені параметри обміну обмежують як список переданої довідкової інформації, так і список документів.

При обміні даними можна регулювати процес вивантаження контактної інформації по партнерам. Процес вивантаження контактної інформації регулюється установкою відповідного значення в полі "Використання контактної інформації".

  • Не використовується - контактна інформація по партнерам не вивантажується.
  • Використовується для читання-вивантажується контактна інформація, зареєстрована в картці партнера і інформація про контактних осіб партнера. Ця інформація може бути використана торговим представником в якості додаткових відомостей про партнера. Торговий представник не може змінювати або реєструвати нову контактну інформацію.
  • Використовується для читання і запису - вивантажується контактна інформація, зареєстрована в картці партнера та інформація про контактних осіб партнера. Ця інформація може бути використана торговим представником в якості додаткових відомостей про партнера. Торговий представник може уточнювати дані щодо контактної інформації партнера і реєструвати нову контактну інформацію по партнерам. Ця інформація при обміні даними буде завантажена в центральну інформаційну базу.

Чи можна зробити так, щоб при завантаженні даних за оформленими завданнями автоматично створювалися замовлення для клієнтів?

Для того щоб при обміні даними створювалися замовлення клієнтам, у формі настройки необхідно встановити прапорець "Автоматично створювати замовлення за завданням". Замовлення клієнта будуть автоматично створені за тими завданнями, у яких встановлено статус "Відпрацьовано". У замовленні заповняться ті позиції номенклатури, для яких заповнене поле "Кількість" у завданні торговому представнику.

Важливо. Дана можливість може бути застосована лише в тому випадку, якщо використовується режим роботи із завданнями торгового представника ( в настройках параметрів обліку в полі "Завдання торговим представникам" встановлено "Використовуються для управління торговими представниками"). Якщо в полі "Завдання торговим представникам" встановлено "Використовуються торговими представниками для планування", то торговий представник створює замовлення клієнтів самостійно і при обміні даними вони завантажуються в центральну інформаційну базу.
