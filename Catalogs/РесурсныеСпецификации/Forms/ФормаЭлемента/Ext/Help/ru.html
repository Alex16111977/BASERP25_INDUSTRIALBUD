Ресурсная спецификация

Ресурсная спецификация описывает поэтапный производственный процесс и ресурсы, необходимые для его осуществления для производства продукции, выполнения работы, осуществления ремонта или разборки.

В рамках выпуска продукции отдельными ресурсными спецификациями описываются переделы, на которых выпускаются полуфабрикаты, входящие в состав продукции.

Спецификации создаются в рабочем месте Производство - Ресурсные спецификации.

Тип производственного процесса
  • Изготовление, сборка - спецификация описывает процесс производства продукции или выполнения работ.
  • Ремонт - описание процесса ремонта.
  • Разборка, утилизация - описывает разборку исходной продукции на компоненты.

Статусы спецификации

Жизненный цикл спецификации описывается следующими статусами:

  • В разработке - черновик спецификации. Могут вноситься любые изменения. Спецификация в этом статусе не должна использоваться в производстве. При возврате спецификации в статус В разработке система предупреждает и предлагает проверить, не используется ли спецификация где-либо;
  • Действует - актуальная действующая спецификация. В этом статусе не допускается правка основной информации спецификации. Для исправления необходимо вернуть в разработку, установив статус В разработке;
  • Закрыта - ранее использовавшаяся спецификация, которая теперь не должна быть доступна для использования. Редактирование закрытых спецификаций не допускается.

Основное

Продукция

Спецификация может быть задана на номенклатуру, характеристику номенклатуры, список номенклатуры или вид номенклатуры.

Если спецификация задана для вида номенклатуры, то есть возможность уточнить для какой номенклатуры и характеристики будет применяться спецификация, в зависимости от свойств продукции.

Если спецификация задана на список номенклатуры, то в табличной части указываются изделия, для производства которых составлена спецификация. Первая строчка является основным изделием и выделяется жирным шрифтом. Остальные изделия - побочные.
Для побочных выходных изделий можно настроить автовыбор и расчет по формуле.
Спецификация доступна для выбора в документах только для тех изделий, где не задана параметризация.

Если основное выходное изделие имеет характеристики, но в спецификации характеристика не указана, то такая спецификация будет подходить для таких изделий с любой характеристикой. Для прочих выходных изделий при наличии характеристик их обязательно уточнять в спецификации.

Пример:
При производстве пластиковой посуды используется пресс-форма для термопласт-автомата, с помощью которой за один раз одновременно отливаются 8 тарелок, 10 ножей, 10 вилок, 10 ложек.
В спецификации в выходных изделиях указывается:

- Тарелка 8 шт
- Нож 10 шт
- Вилка 10 шт
- Ложка 10 шт

Этот комплект из 28 изделий может быть произведен только целиком целое количество раз. Используемая в спецификации и в иных местах единица измерения "Единиц/партий изделий" относится именно к этому комплекту. 5 "Единиц/партий изделий" означает для данной спецификации 40 тарелок, 50 ножей, 50 вилок, 50 ложек.

Параметры назначения спецификаций

При включенной опции НСИ и администрирование - Производство - Использовать параметры назначения спецификаций система позволяет настроить состав и порядок параметров, используемых для подбора подходящих спецификаций. В качестве параметров назначения могут использоваться подразделение-диспетчер, направление деятельности, а также дополнительные реквизиты подразделения-диспетчера и направления деятельности.
Выбранные в настройках для использования параметры назначения появляются в форме спецификации на закладке Основное. Там есть возможность уточнить, для каких именно значений параметров применима спецификация.
При наличии нескольких действующих спецификаций для одного и того же изделия система автоматически подбирает наиболее подходящую спецификацию. При этом спецификация с более уточненными параметрами имеет преимущество перед более общей спецификацией.

Пример
Для изготовления изделия ДСЕ1 имеется 2 действующие спецификации. В спецификации 1 не заданы параметры назначения. В спецификациии 2 в параметрах назначения указано направление деятельности "НД1".
Для заказа на производство без направления деятельности применима только спецификация 1.
Для заказа на производство с направлением деятельности "НД1" применимы обе спецификации, но по умолчанию будет выбрана специфиаци 2, так как она имеет более узкую область применимости.

При наличии нескольких параметров назначения при выборе наиболее подходящей система анализирует порядок параметров. Более высоко расположенный параметр назначения имеет больший приоритет при выборе спецификации.
Если есть несколько спецификаций с одинаковыми параметрами назначения, система выберет самую свежую. То есть такую, которая действует с более поздней даты.

Действующая спецификация может быть исключена из автоматического выбора независимо от параметров назначения. Для этого необходимо на закладке Дополнительно поставить флажок Исключить автоматический выбор в документах. При этом такая спецификация остается доступной для ручного выбора в документах.

Процент брака

В поле "% брака" задается вероятность возникновения брака выходного изделия. Эта настройка трактуется как доля от количества, фактически запущенного в производство. Поэтому, для определения количества, которое необходимо запустить в производство при известной конечной потребности используется формула "Потребность / (1 - %брака / 100)"

Показатель «Процент брака» выходных изделий (продукции) применяется для спецификаций типа «Изготовление, сборка, выполнение работ», «Разборка, утилизация», учитывается при расчете партии производства. Округление при корректировке на процент брака выполняется по математическим правилам (если единица хранения номенклатуры не мерная).

В спецификации "Штамповка подносов" задано, что из одного листа металла делается один поднос. Процент брака равен 10%. Тогда в составе заказа на производство на 10 штук подносов будет задано потребление 11 листов металла.

В спецификации "Штамповка столовых приборов" задано что из листа металла выпускается один нож (брак равен 10%) и одна вилка (брак равен 40%). В заказе на производство на 10 ножей будет задано, что потребуется 11 листов металла. Будет произведено 10 ножей и 7 вилок.

Побочный и промежуточный выпуск

Табличная часть предназначена для побочных и промежуточных выходных изделий, стоимость которых фиксирована и определяется в момент выпуска продукции. Такие изделия уменьшают себестоимость продукции на указанную стоимость. Если изделие не может быть однозначно определено при составлении спецификации, можно настроить автовыбор.

Материалы и работы

В табличной части Материалы и работы представлен перечень материалов, необходимых для производства выходных изделий. В качестве материала может выступать номенклатура с типом Товар, Работа , Набор , Многооборотная тара. Количество материала указывается на все выходные изделия.
Номенклатура с типом Набор добавляется в ресурсную спецификацию в виде отдельной позиции. Далее при формировании документов Этап производства номенклатура распаковывается на комплектующие.

Если материал или его характеристика не могут быть однозначно определены при составлении спецификации, можно настроить их уточнение при использовании спецификации на основании свойств продукции и ее характеристики, либо задать необходимость ручного уточнения. Это делается в настройке автовыбора материала.

Для материала можно настроить потребление в зависимости от свойств продукции.

Способ получения материала

Доступно четыре способа получения материала: обеспечить, обеспечить обособленно, произвести, получить с этапа.

Материал, указанный в спецификации, может являться полуфабрикатом, производимым на этом же предприятии. Если материал является полуфабрикатом, который должен обязательно производиться в процессе производства продукции, необходимо выбрать способ получения материала Произвести или Получить с этапа.

При выборе варианта Произвести надо указать спецификацию, по которой производится этот полуфабрикат.
При выборе варианта Получить с этапа указывается предшествующий этап этой же спецификации.

При использовании спецификации, содержащей производимые в процессе полуфабрикаты, спецификация на продукцию дополняется спецификациями производимых в процессе полуфабрикатов, как если бы они были описаны сразу непосредственно в спецификации на продукцию.

В статусе Действует производится проверка количества потребляемых и производимых на этапах полуфабрикатов.

Применения материала

Если выбрана настройка Применять при производстве по умолчанию, то материалы всегда применяются при планировании запасов и в производстве. Вариант настройки Применять при производстве по требованию позволяет исключить автоматическое добавление материала в производственные документы.

Если указана вероятность применения материала, то планирование потребности в материалах будет выполнено с учетом этой вероятности использования.

По статистике на 10 двигателей в 3 из них Деталь подлежит замене на новую. В спецификации для новой Детали указываем вероятность 30%. При составлении плана производства на ремонт 200 двигателей в план закупок пойдут 60 новых Деталей (или материалов для их изготовления).

Трудозатраты

В табличной части трудозатрат указываются необходимые для производства выходных изделий виды работ сотрудников с указанием количеств.
По кнопке Расчет по формуле можно настроить трудозатраты в зависимости от свойств продукции, аналогично настройке потребления материалов.

Производственный процесс

Производственный процесс спецификации может быть описан одним этапом или несколькими. Для многоэтапного процесса на закладке отображается список этапов. Если требуется только 1 этап, то можно установить переключатель в положение Одноэтапный. Для одноэтапных процессов вместо списка этапов отображается гиперссылка, по которой можно перейти к редактированию этапа.

В многоэтапных спецификациях для материалов, выходных изделий и трудозатрат можно указать этап, на котором они потребляются/возникают. По умолчанию материалы и трудозатраты потребляются на первом этапе, выходные изделия возникают на последнем.

Производственный этап можно выполнять своими силами, либо силами переработчика. Переработка на стороне доступна после включения соответствующей функциональной опции НСИ и администрирование - Производство - Переработка на стороне - Производство силами сторонней организации (переработка на стороне).

Кратность запуска

Параметр Запуск позволяет задать кратность выпуска по ресурсной спецификации. Если задан выпуск кратными партиями или кратно [количество единица измерения] , все, что описано в выходных изделиях, является кратной партией производства по этой спецификации.
Если в поле Запуск выбрано значение произвольными порциями, запуск может быть не кратен выходным изделиям спецификации.

Минимальный запуск

Параметр Минимальный запуск позволяет указать количество, меньше которого запуск нецелесообразен или невозможен.

Оптимальный запуск

Параметр Оптимальный запуск позволяет указать оптимальное количество запуска. Параметр влияет на формирование этапов прозводства. Производимое количество будет разбиваться на партии в соответствии со значением оптимального запуска.
При выборе значения для этого параметра следует понимать, что его уменьшение может привести к увеличению количества формируемых этапов производства.

Допустимое превышение

Допустимое превышение показывает, на сколько возможно увеличить оптимальную партию, если вдруг это потребуется. Параметр может влиять на формирование этапов прозводства.

Партии запуска рассчитываются с учетом трех параметров: оптимальный запуск, минимальный запуск, допустимое превышение.

Пример 1. В заказе 305 штук. Оптимальная партия 100 штук. Допустимое превышение = 6 штук. Минимальная партия = 20.
Получим три партии запуска: 100 шт, 100 шт, 105 шт

Пример 2. В заказе 305 штук. Оптимальная партия 100 штук. Допустимое превышение = 1 штука. Минимальная партия = 20.
Получим четыре партии: 100 штук, 100 штук, 85 штук 20 штук.

Пример 3. В заказе 125 штук, оптимальная партия выпуска равна 100 штукам. Минимальная = 70 штук. Допустимое превышение = 1 штука.
Получим две партии по 70 штук.
